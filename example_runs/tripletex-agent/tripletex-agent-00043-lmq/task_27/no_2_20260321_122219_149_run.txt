### How the accounting entries are generated

When step 4 executes, Tripletex creates these postings automatically:

| Account | Description | Debit (NOK) | Credit (NOK) |
|---|---|---|---|
| 1920 (Bank) | Payment received | 183,603.00 | |
| 1500 (Accounts Receivable) | Clear receivable | | 176,858.40 |
| 8060 (Agio / FX gain) | Exchange rate gain | | 6,744.60 |

### Why no separate voucher is needed

The `PUT /invoice/{id}/:payment` endpoint with **both** `paidAmount` (NOK) and `paidAmountCurrency` (EUR) gives Tripletex all the information it needs to:
1. Debit the bank account for the actual NOK received (183,603.00)
2. Credit the customer's AR for the originally booked NOK amount (176,858.40)
3. Credit the agio account (8060) for the difference (6,744.60)

This is standard Tripletex behavior for foreign currency invoice payments — no manual journal voucher is required.
