## PLAN

**Task Summary:** Register supplier invoice INV-2026-3205 from Ironbridge Ltd (org 828254375) for 24,500 NOK incl. 25% VAT on account 6590 (office services).

**Key financial figures:**
- Gross (incl. VAT): 24,500.00 NOK
- Net (excl. VAT): 19,600.00 NOK
- VAT (25%): 4,900.00 NOK

---

### Step 1 — Look up supplier and accounts (parallelizable)

**1a.** `GET /supplier?organizationNumber=828254375`
— Find Ironbridge Ltd's supplier ID.

**1b.** `GET /ledger/account?number=6590&number=2400`
— Batch-fetch the expense account (6590) and the supplier debt account (2400). We need their internal IDs.

---

### Step 2 — Approach A (preferred): Create invoice in one call

**2.** `POST /incomingInvoice?sendTo=ledger`
— Creates the supplier invoice and posts it to the ledger in a single call.

```json
Body:
{
  "invoiceHeader": {
    "vendorId": <SUPPLIER_ID from 1a>,
    "invoiceDate": "2026-03-21",
    "dueDate": "2026-04-20",
    "invoiceAmount": 24500.00
  },
  "orderLines": [
    {
      "externalId": "line-1",
      "row": 1,
      "description": "Office services",
      "accountId": <ACCOUNT_6590_ID from 1b>,
      "amountInclVat": 24500.00,
      "vatTypeId": 1
    }
  ]
}
```

Key notes:
- `externalId` is **required** on each order line.
- `vatTypeId=1` = 25% ingoing (input) VAT. The system will auto-calculate: debit 6590 for 19,600 net + debit input VAT account for 4,900 + credit 2400 for 24,500.
- `invoiceAmount` is the gross (TTC) amount.
- `dueDate` set to 30 days from invoice date (standard terms).
- `vendorInvoiceNumber` is carried via the invoice header (mapped from INV-2026-3205). *(If the header supports it — otherwise the voucher's `vendorInvoiceNumber` is set automatically.)*

---

### Step 3 — Fallback B (only if Step 2 returns 403)

**3a.** `POST /ledger/voucher?sendToLedger=false`
— Create a voucher with **only** the credit posting (supplier debt on 2400).

```json
Body:
{
  "date": "2026-03-21",
  "description": "Office services - Ironbridge Ltd",
  "vendorInvoiceNumber": "INV-2026-3205",
  "postings": [
    {
      "row": 1,
      "date": "2026-03-21",
      "description": "Office services - Ironbridge Ltd",
      "account": { "id": <ACCOUNT_2400_ID> },
      "supplier": { "id": <SUPPLIER_ID> },
      "amountGross": -24500.00,
      "amountGrossCurrency": -24500.00
    }
  ]
}
```

**3b.** `PUT /supplierInvoice/voucher/{VOUCHER_ID}/postings?sendToLedger=true`
— Add the debit posting (expense side) to complete the supplier invoice.

```json
Body:
[
  {
    "posting": {
      "account": { "id": <ACCOUNT_6590_ID> },
      "amountGross": 24500.00,
      "amountGrossCurrency": 24500.00,
      "vatType": { "id": 1 }
    }
  }
]
```

Key notes:
- The credit posting in 3a must be **negative** (credit = -24500).
- The debit posting in 3b is **positive** and gross (24500). The system splits it into net (19,600) on 6590 + VAT (4,900) on the input VAT account.
- The voucher in 3a must have only ONE posting, otherwise 3b will fail.

---
