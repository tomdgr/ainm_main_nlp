## PLAN

### Phase 1: Parallele Lookups (alle unabhängig voneinander)

**1a.** `GET /invoice?invoiceDateFrom=2025-01-01&invoiceDateTo=2026-03-21`
→ Alle Rechnungen abrufen, um die **überfällige Rechnung** zu identifizieren (wo `invoiceDueDate < 2026-03-21` und offener Restbetrag > 0). Daraus ergibt sich die **overdue_invoice_id** und die **customer_id**.

**1b.** `GET /ledger/account?number=1500&number=3400&number=1920&fields=id,number,version,bankAccountNumber`
→ Alle drei Konten in einem Call abrufen:
- **1500** (Forderungen) — für die Buchung (Soll)
- **3400** (Mahngebühren) — für die Buchung (Haben)
- **1920** (Bankkonto) — prüfen, ob `bankAccountNumber` gesetzt ist

**1c.** `GET /invoice/paymentType`
→ Zahlungsarten abrufen, um die passende **paymentTypeId** für die Teilzahlung (Bankkonto) zu finden.

**1d.** `GET /ledger/voucherType`
→ Belegarten abrufen, um den richtigen Typ für den manuellen Buchungsbeleg zu finden (z.B. "Manuelt bilag" o.ä.).

**1e.** `GET /ledger/vatType?vatDate=2026-03-21`
→ MwSt-Typen abrufen, um den **MwSt-freien Typ** für die Mahngebühr zu finden (Purregebyr ist in Norwegen umsatzsteuerfrei, z.B. Kode 0 oder 6 = "Ingen mva").

### Phase 3: Mahngebühr buchen (Soll 1500 / Haben 3400)

**3.** `POST /ledger/voucher?sendToLedger=true`
→ Buchungsbeleg für die Mahngebühr von 70 NOK erstellen.
```json
{
  "date": "2026-03-21",
  "description": "Purregebyr / Mahngebühr",
  "voucherType": {"id": <voucherType_id>},
  "postings": [
    {
      "date": "2026-03-21",
      "description": "Purregebyr",
      "account": {"id": <1500_id>},
      "customer": {"id": <customer_id>},
      "amount": 70
    },
    {
      "date": "2026-03-21",
      "description": "Purregebyr",
      "account": {"id": <3400_id>},
      "amount": -70
    }
  ]
}
```
- `amount: 70` (positiv) = **Soll** auf Konto 1500 (Forderungen)
- `amount: -70` (negativ) = **Haben** auf Konto 3400 (Mahngebühren/Erträge)
- `customer` wird nur an der Soll-Buchung (1500) hinterlegt, um die Forderung dem Kunden zuzuordnen

---

### Phase 4: Rechnung über die Mahngebühr erstellen & senden

**4.** `POST /order`
→ Auftrag für die Mahngebühr-Rechnung anlegen.
```json
{
  "customer": {"id": <customer_id>},
  "deliveryDate": "2026-03-21",
  "orderDate": "2026-03-21",
  "orderLines": [
    {
      "description": "Purregebyr / Mahngebühr",
      "unitPriceExcludingVatCurrency": 70,
      "count": 1,
      "vatType": {"id": <no_vat_type_id>}
    }
  ]
}
```
- Kein `product` nötig — `description` + `unitPriceExcludingVatCurrency` reichen aus
- `vatType` = MwSt-frei, da Mahngebühren in Norwegen umsatzsteuerfrei sind

**5.** `PUT /order/{new_order_id}/:invoice?invoiceDate=2026-03-21&sendToCustomer=true`
→ Rechnung aus dem Auftrag erstellen **und direkt an den Kunden senden** (ein Schritt).

---

### Phase 5: Teilzahlung auf die überfällige Rechnung buchen

**6.** `PUT /invoice/{overdue_invoice_id}/:payment?paymentDate=2026-03-21&paymentTypeId=<bank_payment_type_id>&paidAmount=5000`
→ **5.000 NOK Teilzahlung** auf der überfälligen Rechnung registrieren.
- `paymentTypeId` = ID des Bank-Zahlungstyps (aus Schritt 1c)
- `paidAmount` = 5000

### Zusammenfassung der Abhängigkeiten

```
Phase 1: [1a, 1b, 1c, 1d, 1e] — parallel
                ↓
Phase 2: [2] — bedingt (nur wenn 1920 kein bankAccountNumber hat)
                ↓
Phase 3: [3] — braucht: account-IDs (1b), voucherType (1d), customer_id (1a)
                ↓
Phase 4: [4 → 5] — braucht: customer_id (1a), vatType (1e), bank-Check (2)
                ↓
Phase 5: [6] — braucht: overdue_invoice_id (1a), paymentTypeId (1c)
```
