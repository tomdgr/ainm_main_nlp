### Forklaring av flyten:

| Steg | Formål | Avhengigheter |
|------|--------|---------------|
| **1–2** | Parallelle oppslag — finn avdelings-ID og stillingskode-ID | Ingen (kan kjøres samtidig) |
| **3** | Opprett ansatt med personnummer, fødselsdato, avdeling | Avhenger av steg 1 (department ID) |
| **4** | Opprett arbeidsforhold med lønn, stillingsprosent, startdato og stillingskode — `employmentDetails` legges inn **inline** for å spare ett API-kall | Avhenger av steg 2 (occupationCode ID) og steg 3 (employee ID) |

### Nøkkelbeslutninger:
- **4 kall totalt** (2 oppslag + 2 opprettelser) — optimalt fordi `employmentDetails` bakes inn i `POST /employee/employment`
- **`startDate`** settes på employment-objektet, *ikke* på employee (vanlig feilkilde)
- **`nationalIdentityNumber`** brukes for personnummer (11-sifret)
- **`userType: "NO_ACCESS"`** brukes med mindre kontrakten tilsier systemtilgang
- **`taxDeductionCode: "loennFraHovedarbeidsgiver"`** — standard for hovedarbeidsgiver
- Ingen verifiserings-GET etter opprettelse — 201-responsen bekrefter suksess
