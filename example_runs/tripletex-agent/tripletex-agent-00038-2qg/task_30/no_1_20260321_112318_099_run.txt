### Resumo do fluxo e dependências

```
Passo 1 ─┐
Passo 2 ─┤ (paralelos — sem dependências entre si)
          │
          ▼
Passos 3, 4, 5, 6  (dependem dos IDs obtidos em 1+2;
                     podem ser paralelos entre si)
          │
          ▼
Passo 7   (DEVE esperar por 3-6 para que o razão inclua
           as depreciações e a reversão nos saldos)
          │
          ▼
Passo 8   (depende do resultado calculado em 7)
```

| Fase | Chamadas | Propósito |
|------|----------|-----------|
| 1 | 2 GET | Resolver IDs (contas + tipo voucher) |
| 2 | 3 POST | Registar 3 depreciações separadas |
| 3 | 1 POST | Reverter despesas antecipadas |
| 4 | 1 GET | Ler razão para calcular resultado tributável |
| 5 | 1 POST | Registar provisão fiscal |
| **Total** | **8 chamadas API** | |

### Notas importantes

- **Montantes positivos = débito**, **negativos = crédito** na API `POST /ledger/voucher`
- Cada voucher é auto-balanceado (soma dos postings = 0)
- A conta **7700** (Annen driftskostnad / Outras despesas operacionais) é usada para a reversão de antecipadas, pois o enunciado não especifica uma conta de despesa particular
- O passo 7 é **sequencialmente obrigatório** antes do 8, pois precisamos do resultado já com depreciações e reversão incluídas para calcular correctamente os 22%
- Todos os valores são arredondados a 2 casas decimais conforme exigido pela API
