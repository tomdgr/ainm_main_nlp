## ✅ PLAN: Registrer leverandørfaktura INV-2026-8584 via bilag (ledger voucher)

**Bakgrunnsberegning (25 % MVA):**
- Bruttobeløp (inkl. MVA): **11 950,00 kr**
- Nettobeløp (ekskl. MVA): 11 950 / 1,25 = **9 560,00 kr**
- MVA-beløp: 11 950 − 9 560 = **2 390,00 kr**

---

### Steg 1 — Finn leverandøren
```
GET /supplier?organizationNumber=852796316
```
→ Henter Snøhetta AS sin leverandør-ID (`supplierId`).

---

### Steg 2 — Finn MVA-type, kostnadskonto og bilagstype (3 parallelle kall)
```
GET /ledger/vatType?number=1
```
→ Henter ID for inngående MVA 25 % («Fradrag inngående avgift, høy sats»).

```
GET /ledger/account?number=7000
```
→ Verifiserer at konto 7000 (kontortjenester) finnes og henter dens ID.

```
GET /ledger/voucherType?name=Leverandør
```
→ Henter bilagstype-ID for leverandørfaktura.

---

```
POST /ledger/voucher?sendToLedger=true
```

| Rad | Konto | Debet/Kredit | Bruttobeløp | MVA-type | Kommentar |
|-----|-------|-------------|-------------|----------|-----------|
| 1 | 7000 | Debet | 11 950,00 | Inng. MVA 25 % (id fra steg 2) | Tripletex splitter auto: 9 560 på 7000 + 2 390 på MVA-konto |
| 2 | 2400 | Kredit | −11 950,00 | *(ingen)* | Leverandørgjeld, knyttet til `supplier.id` |

**Viktige detaljer:**
- `amountGross` og `amountGrossCurrency` settes **likt** (NOK → NOK).

- Kredit-linje (konto 2400) kobles til leverandøren via `supplier: {id}` for å opprette åpen post.
- `vendorInvoiceNumber` sikrer sporbarhet til fakturareferansen INV-2026-8584.
