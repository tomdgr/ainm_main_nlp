## PLAN

### Step 1 — Fetch January & February postings + find an existing activity *(3 parallel calls)*

**1a.** `GET /ledger/postingByDate?dateFrom=2026-01-01&dateTo=2026-02-01&count=5000&fields=account(id,name,number),amount`
— Retrieve all general ledger postings for **January 2026** (`dateTo` is exclusive, so 2026-02-01 correctly caps at end of January). Using `fields` filter to return only account info and amounts, minimising payload.

**1b.** `GET /ledger/postingByDate?dateFrom=2026-02-01&dateTo=2026-03-01&count=5000&fields=account(id,name,number),amount`
— Same for **February 2026**.

**1c.** `GET /activity?isProjectActivity=true&count=1`
— Retrieve at least one existing activity (needed to link to each `ProjectActivity`). The `isProjectActivity=true` filter ensures it is usable in a project context.

---

### Step 2 — Analysis *(no API call)*

Using the postings from Steps 1a & 1b:
- Group postings by `account.number` / `account.name`.
- Sum the `amount` per account for each month (expense accounts typically numbered **4000–8999** in the Norwegian chart of accounts).
- Compute the **increase = February total − January total** for each account.
- Rank by largest positive increase → select the **top 3 expense accounts**, capturing their `name`.

---

### Step 3 — Create 3 internal projects with embedded activities *(1 batch call)*

**`POST /project/list`**
— Create all three projects in a single batch request. Each project embeds a `projectActivities` entry so no separate activity-creation call is needed.

```json
[
  {
    "name": "<Account 1 Name>",
    "isInternal": true,
    "startDate": "2026-03-21",
    "projectActivities": [
      { "activity": { "id": "<activity_id from Step 1c>" } }
    ]
  },
  {
    "name": "<Account 2 Name>",
    "isInternal": true,
    "startDate": "2026-03-21",
    "projectActivities": [
      { "activity": { "id": "<activity_id from Step 1c>" } }
    ]
  },
  {
    "name": "<Account 3 Name>",
    "isInternal": true,
    "startDate": "2026-03-21",
    "projectActivities": [
      { "activity": { "id": "<activity_id from Step 1c>" } }
    ]
  }
]
```

| # | Call | Purpose |
|---|------|---------|
