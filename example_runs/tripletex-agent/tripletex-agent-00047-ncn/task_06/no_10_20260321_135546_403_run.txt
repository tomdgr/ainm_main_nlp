## 🇳🇴 Norwegian Accounting Rules Applied

| Element | Decision | Reason |
|---|---|---|
| **Expense account** | **7350** – *Kaffemøte og representasjon* | Standard Norwegian NS 4102 account for coffee meetings/representation |
| **VAT treatment** | **No input VAT deduction** | Kaffemøte is classified as *representasjon* — MVA-loven § 8-3 explicitly excludes entertainment/representation from input VAT recovery |
| **Posting method** | Full **gross amount** to account 7350 | Since VAT is non-deductible, the total price (incl. 25% VAT) is the full expense — no VAT account split |
| **Credit account** | **1920** – *Bankinnskudd* | Receipt implies payment was already made |
| **Department tag** | HR department on the debit posting | As specified |

## ✅ EXECUTION PLAN

```
STEP 1 — GET /department?name=HR
  Purpose: Retrieve the HR department's internal ID for use in the posting

STEP 2 — GET /ledger/account?number=7350&number=1920
  Purpose: Confirm both accounts exist in this company's chart of accounts
           in one single call
           • 7350 = Kaffemøte/representasjon (debit/expense side)
           • 1920 = Bank (credit/payment side)

STEP 3 — GET /ledger/vatType
  Purpose: Find the VAT type corresponding to non-deductible representation
           (typically labelled "Representasjon" or "Ingen fradragsrett",
           typeOfVat filter may be applied). This ID is required in
           Tripletex even when deductibility is 0%, to correctly flag
           the transaction for VAT reporting purposes.

STEP 4 — POST /ledger/voucher?sendToLedger=true
  Purpose: Create and immediately send the Kaffemøte voucher to the ledger
  Body:
  {
    "date": "2026-03-18",
    "description": "Kaffemøte",
    "postings": [
      {
        // DEBIT — Expense side
        "account":     { "id": <id of account 7350> },
        "department":  { "id": <id of HR department> },
        "amountGross": <gross total from receipt>,
        "vatType":     { "id": <id of non-deductible representation VAT type> },
        "description": "Kaffemøte"
      },
      {
        // CREDIT — Payment side (already paid from bank)
        "account":     { "id": <id of account 1920> },
        "amountGross": <same gross total, negative>,
        "description": "Kaffemøte"
      }
    ]
  }
```

---

## 📌 Key Notes for the Executor

1. **`sendToLedger=true`** — ensures the voucher is not left as a draft; it flows immediately into the GL.
2. **Gross amount only** — the API documentation explicitly states *"Only the gross amounts will be used"* — do NOT attempt to split net + VAT into separate amounts.
3. **VAT type is still required** — even though deductibility is 0%, you must attach the non-deductible VAT type (found in Step 3) so that Tripletex correctly flags the transaction for VAT return reporting (MVA-melding).
4. **Department tag on the debit posting only** — it is standard practice to tag the expense side; the bank credit posting does not need a department.
5. **Amounts from the receipt** — the actual NOK gross total must be read from the PDF before executing Step 4.
