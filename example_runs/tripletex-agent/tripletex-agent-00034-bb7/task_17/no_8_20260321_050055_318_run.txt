## PLANO DE EXECUÇÃO

### Contexto
Criar uma dimensão contabilística personalizada "Marked" com dois valores ("Offentlig" e "Bedrift"), e depois lançar um voucher na conta 6540 por 44 100 NOK vinculado ao valor "Offentlig".

---

### Etapa 1 — Criar a dimensão contabilística "Marked"

```
1. POST /ledger/accountingDimensionName — Criar a dimensão personalizada "Marked"
   Body:
   {
     "dimensionName": "Marked",
     "description": "Marked",
     "active": true
   }
```
> A resposta devolverá o `id` e, implicitamente, o `dimensionIndex` que precisamos para criar os valores.

---

### Etapa 2 — Obter o dimensionIndex da dimensão criada

```
2. GET /ledger/accountingDimensionName/search — Buscar a dimensão "Marked" para obter o dimensionIndex
   (Usa o ID retornado no passo 1 ou busca por nome para confirmar o dimensionIndex)
```
> Alternativa: usar `GET /ledger/accountingDimensionName/{id}` com o ID do passo 1.

---

### Etapa 3 — Criar os dois valores da dimensão

```
3. POST /ledger/accountingDimensionValue — Criar valor "Offentlig"
   Body:
   {
     "displayName": "Offentlig",
     "number": "1",
     "dimensionIndex": <dimensionIndex do passo 2>,
     "active": true,
     "showInVoucherRegistration": true
   }

4. POST /ledger/accountingDimensionValue — Criar valor "Bedrift"
   Body:
   {
     "displayName": "Bedrift",
     "number": "2",
     "dimensionIndex": <dimensionIndex do passo 2>,
     "active": true,
     "showInVoucherRegistration": true
   }
```
> Estes dois calls podem ser feitos em **paralelo** pois são independentes. Não existe endpoint `/list` para `accountingDimensionValue`, portanto são 2 chamadas separadas.
> O passo 3 retorna o `id` do valor "Offentlig", necessário para o voucher.

---

### Etapa 4 — Buscar a conta 6540

```
5. GET /ledger/account?number=6540 — Buscar o ID da conta contabilística 6540
```
> Pode ser executado em paralelo com os passos 3 e 4.

---

### Etapa 5 — Buscar o tipo de voucher adequado

```
6. GET /ledger/voucherType?name=Manualpostings — Buscar o tipo de voucher para lançamento manual
   (Se não encontrar, tentar sem filtro para listar os tipos disponíveis)
```
> Pode ser executado em paralelo com os passos 3, 4 e 5.

---

### Etapa 6 — Lançar o voucher na conta 6540 com dimensão "Offentlig"

```
7. POST /ledger/voucher?sendToLedger=true — Criar voucher com postings
   Body:
   {
     "date": "2026-03-21",
     "description": "Lançamento conta 6540 - Dimensão Marked: Offentlig",
     "voucherType": { "id": <voucherType_id do passo 6> },
     "postings": [
       {
         "date": "2026-03-21",
         "description": "Lançamento Offentlig",
         "account": { "id": <account_id_6540 do passo 5> },
         "amount": 44100.00,
         "freeAccountingDimension1": { "id": <id_Offentlig do passo 3> }
       },
       {
         "date": "2026-03-21",
         "description": "Contrapartida",
         "account": { "id": <account_id_contrapartida> },
         "amount": -44100.00
       }
     ]
   }
```

---

| 2 | `GET` | `/ledger/accountingDimensionName/{id}` | Obter `dimensionIndex` |
| 3 | `POST` | `/ledger/accountingDimensionValue` | Criar valor "Offentlig" |
| 4 | `POST` | `/ledger/accountingDimensionValue` | Criar valor "Bedrift" |
| 5 | `GET` | `/ledger/account?number=6540` | Buscar conta 6540 |
| 6 | `GET` | `/ledger/voucherType` | Buscar tipo de voucher |

### Notas Importantes
- O campo `freeAccountingDimension1` corresponde à 1ª dimensão livre. Se a dimensão "Marked" for atribuída a outro índice (e.g., dimensionIndex 2), usar `freeAccountingDimension2`.

- Os passos 3, 4, 5 e 6 podem correr **em paralelo** após o passo 2.
